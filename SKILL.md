# dbs-benchmark R3 — 对标分析

> 完整原始文档: `references/original_body.md` (13.6KB)

## 一句话定义

帮用户找到值得模仿的对标，用五重过滤法排除一切干扰——模仿是正确路径，但模仿谁决定模仿的质量。

---

## 4 条核心信条

| # | 信条 |
|---|------|
| 1 | **排除自我是决策加速器**：不要想"我能不能做"，先看"这个模式能不能跑"。自我意识是对标分析的最大噪音 |
| 2 | **N+1 阶段**：找比你高一级的对标，不要找高出 10 级的。高出 10 级你只能看到结果看不到过程 |
| 3 | **模仿的颗粒度决定模仿的质量**：不是模仿"他在做知识付费"，是模仿"他每周发 3 篇 800 字文章，每篇结尾挂 99 元课程链接" |
| 4 | **高利润是唯一标准**：只对标能验证高利润的对象。粉丝量、点赞数、知名度——全是干扰。看不到利润数据的对标=无效对标 |

---

## 5 Phase 对标流程

### Phase 1：搞清楚用户现在的状态
问：你现在有什么？产品、内容、客户、收入——四个维度各描述一句。

### Phase 2：确定对标域
根据 Phase 1 的状态，确定 N+1 的对标域：
- 无产品 → 对标有产品且刚起步的人
- 有产品无客户 → 对标有客户且客户获取方式可复制的人
- 有客户无利润 → 对标高利润模式（不是大流量模式）

### Phase 3：五重过滤

| 过滤层 | 过滤内容 | 排除信号 |
|--------|---------|---------|
| 第一重 | 不可验证利润 | 没公开过收入/无第三方数据 |
| 第二重 | 高出 N+2 以上 | 规模远超当前阶段 |
| 第三重 | 模式不可复制 | 靠资源/关系/运气而非方法论 |
| 第四重 | 颗粒度太粗 | 只能说出"他在做 XX"说不出具体动作 |
| 第五重 | 用户已有偏见 | 用户说"不可能是他"但给不出证据 |

### Phase 4：输出对标画像
```
# 对标画像：{对标名称}

## 基本信息（谁/做什么/多少年）
## 产品（卖什么/多少钱/怎么卖）
## 客户（客户是谁/从哪来/为什么买）
## 可验证利润信号（公开数据/第三方估算/保守下限）
## 可模仿的具体动作（≤5条，颗粒度到每周做什么）
## 和他不一样的 1 件事（找到差异化空间）
```

### Phase 5：模仿行动计划
基于对标画像给出 ≤3 条模仿动作，每条含：
- 具体做什么（动词开头 + 数字 + 频率）
- 模仿的颗粒度（到什么程度算模仿到位）
- 第一个里程碑（什么时候算有进展）

```
dbs-benchmark ──→ 对标明确后:
      ├──→ dbs-action (知道对标但不想模仿)
      ├──→ dbs-content (对标的内容策略需要分析)
      ├──→ dbs-slowisfast (需要对标的是慢方法)
      └──→ dbs-decision (在多个对标中选择)
```

## 验证清单

- [ ] 用户当前状态四维度已描述
- [ ] N+1 对标域已界定
- [ ] 五重过滤全部通过
- [ ] 对标画像含可验证利润信号
- [ ] 模仿动作含具体数字/频率
- [ ] 差异化空间已标注

## 失败模式与兜底

| 失败 | 兜底 |
|------|------|
| 用户选的对标高出 N+3 以上 | 指出阶段差距，建议降级到 N+1 |
| 对标无法验证利润 | 标记为"不可验证对标"，建议找替代 |
| 用户拒绝所有建议的对标 | 追问真实原因——可能是心理抗拒(→dbs-action) |
